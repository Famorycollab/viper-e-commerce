# Corriger les bugs du site

Tu travailles sur une application frontend Vite + React + TypeScript + Tailwind CSS pour un site e-commerce local.

## Objectif
Corriger les bugs signalés ou observables sur le site en identifiant la cause racine, en appliquant une correction minimale et en vérifiant le résultat.

## Contexte du projet
- L’application est une SPA front-only sans backend.
- L’état du panier, de la wishlist et du catalogue est géré localement dans le contexte React.
- Les données produits sont centralisées dans src/data/products.ts.
- Le stockage persistant se fait via localStorage.
- Les fichiers principaux à consulter sont :
  - src/App.tsx
  - src/context/CartContext.tsx
  - src/data/products.ts
  - src/index.css
  - src/main.tsx

## Processus à suivre
1. Comprendre le bug observé ou décrit.
2. Reproduire le problème si possible, ou l’analyser à partir du code et du comportement attendu.
3. Identifier la cause racine avant toute modification.
4. Appliquer une correction ciblée, simple et robuste.
5. Vérifier que le correctif ne casse pas les autres comportements du site.
6. Expliquer clairement ce qui a été corrigé et comment il a été validé.

## Règles importantes
- Respecter l’architecture existante : React + TypeScript, état local, contexte React, localStorage.
- Éviter d’introduire un backend ou une API si ce n’est pas demandé.
- Préserver les labels UI en français et les sections principales du site.
- Éviter les solutions bricolées : privilégier une correction propre et durable.
- Ne pas utiliser any si une solution typée est possible.
- Si un comportement dépend de l’état du navigateur ou du stockage local, tenir compte de ce contexte.

## Format de sortie attendu
Fournis un résumé clair avec :
- Le bug corrigé ou le problème traité.
- La cause racine identifiée.
- Les fichiers modifiés.
- La vérification effectuée (ex. build, tests, contrôle manuel si pertinent).
- Si nécessaire, les limites ou points à surveiller.

## Exemple d’invocation
- Corrige les bugs visibles sur la page boutique.
- Répare un problème de panier qui ne se met pas à jour correctement.
- Corrige un affichage cassé sur mobile.

## À faire si l’information manque
Si le bug n’est pas clairement défini, demande une précision avant d’éditer le code :
- Quel comportement est cassé ?
- Sur quelle page ou action ?
- Est-ce un bug de rendu, d’interaction, de persistance ou de style ?
